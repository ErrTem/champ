## Phase 05 Screen Contracts

### My bookings (list) ÔÇö MBB-01

- **Information architecture**: 2 tabs ÔÇö **Upcoming** and **Past**.
- **Sorting**:
  - Upcoming: soonest-first
  - Past: newest-first
- **Row content** (compact scan-friendly):
  - Fighter name (primary)
  - Service title (secondary)
  - Local date/time (America/Los_Angeles display, no timezone label)
  - Status chip
- **Interactions**:
  - Tap row Ôćĺ booking detail
  - Pull-to-refresh optional (recommended)
- **Loading**: skeleton rows (no spinners-only pages).
- **Empty state**:
  - Friendly message + CTA **Explore fighters**.
  - CTA routes to the existing fighter discovery experience (catalog/home).
- **Error state**:
  - Inline card (glass/tonal) with error copy and **Retry** button using `.kinetic-gradient`.

Source: `.planning/phases/05-my-bookings-notifications/05-CONTEXT.md` (D-01..D-04, D-06)

### Booking detail ÔÇö MBB-02

- **Fields shown**: fighter, service, local time, booking status, payment state, price.
- **Actions**:
  - If `awaiting_payment`: primary CTA **Pay now** (accent gradient).
  - For terminal statuses (past/confirmed/expired): **no actions**.
  - No cancel action in Phase 05.
- **Time display**:
  - Display in **Pacific Time local time only**
  - Do **not** add a timezone label.
- **Status presentation**:
  - Status chip uses tonal background (no borders). Reserve full accent only for the primary CTA.

Source: `.planning/phases/05-my-bookings-notifications/05-CONTEXT.md` (D-05..D-08, D-19)
