<task type="auto">
  <name>Task 2: Implement Google OAuth strategy + routes</name>
  <files>backend/src/auth/strategies/google.strategy.ts,backend/src/auth/auth.controller.ts,backend/src/auth/auth.module.ts</files>
  <action>
Use `passport-google-oauth20` strategy:
- `GET /auth/google` starts flow
- `GET /auth/google/callback` handles provider callback

On callback success:
- find or create `User` + `UserOAuthIdentity`
- issue cookies using existing cookie helper
- redirect to frontend route (e.g. `/profile` or `/explore`) with safe allowlisted relative path param.

Doc reference: [CITED: https://www.passportjs.org/packages/passport-google-oauth20/]
  </action>
  <acceptance_criteria>
    - `GET /auth/google` redirects to Google consent screen.
    - `GET /auth/google/callback` sets cookies and redirects to safe in-app path.
  </acceptance_criteria>
  <verify>
    <automated>backend: npm test</automated>
  </verify>
  <done>
    Google OAuth flow creates/links user identity, sets session cookies, and redirects only to safe in-app path.
  </done>
</task>
