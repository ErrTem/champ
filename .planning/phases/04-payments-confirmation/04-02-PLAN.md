## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| client → API | Client requests checkout; must not be able to tamper with amount or booking ownership. |
| API → Stripe | Server creates Checkout Session with authoritative amount and minimal metadata. |
| API → DB | Session id persisted to enforce D-04 and support webhook correlation. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | Checkout amount | mitigate | Compute `unit_amount` from DB `Service.priceCents` for booking’s service (D-06); ignore client-provided values; cover with backend test. |
| T-04-06 | Spoofing | Booking ownership | mitigate | Ensure bookingId lookup is scoped to `userId = req.user.sub`; return 404/403 on mismatch; cover with backend test. |
| T-04-07 | Information Disclosure | Stripe metadata | mitigate | Only `metadata.bookingId` allowed (D-05); do not attach user email/phone/name; avoid logging Stripe session object. |
</threat_model>
