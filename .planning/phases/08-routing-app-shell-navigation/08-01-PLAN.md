<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| user-controlled URL → Angular Router | path + query params (including `returnTo`) are untrusted input |
| unauthenticated user → guarded routes | access attempts to `/bookings/**`, `/profile`, `/admin/**` |
| authenticated non-admin → admin UI/routes | must not gain admin access; UI must not leak admin affordances by default |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-08-01 | S | `LoginPage` `returnTo` redirect | mitigate | Enforce open-redirect safety: accept `returnTo` only when string starts with `/` and not `//`; else ignore and go `/explore` (per UI-SPEC). |
| T-08-02 | E | Admin navigation affordance | mitigate | Hide Admin tab unless `user.isAdmin===true`; keep `adminGuard` on `/admin/**`; while profile loading/unknown, hide tab (secure default). |
| T-08-03 | I | Exposing admin-only nav to non-admin | mitigate | Ensure TabsPage conditionally renders Admin tab; ensure `/admin/**` non-admin access redirects away via guard (existing). |
| T-08-04 | T | Incorrect redirect rules (landing on removed Home) | mitigate | Replace `'' -> home` with `'' -> explore`; add wildcard `** -> explore`; remove/disable Home route as entrypoint. |
</threat_model>
