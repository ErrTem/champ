<threat_model>
| Threat | Category | Mitigation |
|--------|----------|------------|
| Push subscription tampering (user binds чужой endpoint to self) | Tampering | Upsert requires auth; endpoint treated opaque; ownership enforced by reassigning endpoint to caller only if caller proves control via browser (practical); log reassignment; optional: reject reassignment unless previous owner same user. |
| Subscription enumeration via IDs | Info disclosure | Use cuid ids + require ownership; prefer 404 on чужой delete; do not expose endpoints for other users. |
| Unauthorized ICS access (guess booking id) | Info disclosure | AuthZ: only booking customer, fighter, admin; prefer 404 for unauthorized; rate limit endpoint; avoid including PII beyond needed. |
| Abuse/spam: attacker creates many subs and forces sends | Abuse | Cap subscriptions per user (soft limit e.g. 10) + cleanup invalid endpoints; later add rate limiting on upsert. |
</threat_model>
