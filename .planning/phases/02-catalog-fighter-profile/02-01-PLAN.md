Add a new Nest module `FightersModule` that exposes two public endpoints:

1) `GET /fighters`
- Returns only fighters where `published=true`.
- Returns fields: `id`, `name`, `photoUrl`, `summary`, `disciplines` (if modeled), plus `fromPriceCents` computed as the minimum `priceCents` across that fighter’s **published** services.
- If a fighter has no published services, set `fromPriceCents: null` (and still include fighter only if product chooses; default is still include but UI can show “From —”).

2) `GET /fighters/:fighterId`
- If fighter is unpublished or missing → 404.
- Returns: fighter profile fields (`id`, `name`, `photoUrl`, `bio`, `disciplines`, `mediaUrls`), required stats (wins/losses/draws/yearsPro), and `services[]` containing only `published=true` services with `id`, `title`, `durationMinutes`, `modality`, `priceCents`, `currency`.

Implementation constraints:
- Do NOT require auth/cookies; no guards.
- Do NOT return raw Prisma entities; map to explicit DTOs.
- Ensure published filtering applies to both fighters and services and also to the min-price computation (“From $X”).
- Avoid N+1: implement either include services and compute in memory, or groupBy `_min` then join.

Wire `FightersModule` into `backend/src/app.module.ts`.
